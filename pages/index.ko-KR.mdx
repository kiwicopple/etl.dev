# ETL: 추출, 변환, 로드

컴퓨팅에서 추출, 변환, 로드는 데이터가 소스로부터 추출되어 변환되고 새로운 대상으로 로드되는 세 단계 프로세스입니다.

## ETL (추출, 변환, 로드)이란?

ETL은 추출, 변환, 로드를 나타내는 용어입니다. 데이터 웨어하우징에서 사용되는 프로세스로, 다음을 포함합니다:

- 다양한 소스 시스템에서 데이터를 **추출**합니다.
- 데이터를 분석에 적합한 형식으로 **변환**합니다.
- 최종 대상 데이터베이스, 데이터 웨어하우스 또는 데이터 레이크에 **로드**합니다.

## 데이터 처리에서 ETL의 중요성

ETL은 데이터 기반 의사 결정에 있어서 중요합니다. 이는 다음과 같은 이유로 인해 중요합니다:

- 다양한 데이터를 통합하여 통일된 형식으로 유지합니다.
- 데이터의 품질과 정확성을 보장합니다.
- 효율적인 데이터 저장 및 분석을 용이하게 합니다.

## ETL 워크플로우 개요

전형적인 ETL 워크플로우는 세 가지 단계로 구성됩니다:

1. **추출**: 여러 소스 시스템에서 데이터를 수집합니다. 이 데이터는 형식과 구조가 다양할 수 있습니다.
2. **변환**: 추출된 데이터는 대상 스키마에 맞게 클린징, 유효성 검사 및 변환을 거칩니다.
3. **로드**: 변환된 데이터는 데이터 웨어하우스로 전송되어 저장 및 분석됩니다.

## ETL에서의 주요 개념

ETL에서 이해해야 할 주요 개념은 다음과 같습니다:

- **데이터 통합**: 다른 소스에서 데이터를 결합하여 통합된 뷰를 생성합니다.
- **데이터 정제**: 부정확성을 수정하여 데이터 품질을 향상시킵니다.
- **데이터 변환**: 데이터를 한 형식 또는 구조에서 다른 형식 또는 구조로 변환합니다.
- **일괄 처리 vs. 실시간 처리**: 대량의 데이터를 처리할 것인지 연속적이고 실시간으로 처리할 것인지 결정합니다.
